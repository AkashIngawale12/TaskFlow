# TaskFlow: Built 100% with Cursor AI

**This entire iOS project—from idea creation to the full codebase—was built using Cursor AI only. No manual coding was used.**  
The workflow below is the exact process used so others can replicate the approach.

---

## Summary

- **Idea generation** → done with AI (e.g. ChatGPT) to get app options.
- **Cursor build prompt** → generated by AI from a short brief so the long prompt wasn’t written by hand.
- **Full app implementation** → built in Cursor by pasting the generated prompt; all code, structure, and features came from Cursor.

No code in this repository was written manually; all Swift, SwiftUI, SwiftData, views, view models, and tests were produced by Cursor in response to the prompts described below.

---

## End-to-End Workflow

### Step 1: Generate iOS app ideas

**What we did:** Asked an AI to suggest 5–8 strong app ideas in a chosen category (e.g. task management), then picked one (TaskFlow).

**Prompt used:**

```
You are a senior iOS developer and architect

Generate 5–8 strong mobile app ideas for the following category:

CATEGORY: Task management

Focus on real problems, underserved niches, and differentiated products.

For each idea provide:

1. App Name

2. Concept
   - What the app does
   - Target users
   - Problem it solves

3. Why This Idea Could Succeed
   - What makes it unique

4. Core Features
   - 4–6 meaningful features

5. Technology Opportunities
   - Key technologies that could power the app

Ensure ideas feel like real apps that could launch on the App Store.
```

**Outcome:** Several options (e.g. TaskFlow, Recipe Vault, Expense Tracker). We chose **TaskFlow**.

---

### Step 2: Generate the Cursor prompt

**What we did:** Used an AI tool (e.g. ChatGPT) to turn a short brief into a single, ready-to-use Cursor prompt so we didn’t write the long prompt manually.

**Prompt used:**

```
You are a senior iOS engineer. Generate a single, ready-to-use Cursor prompt that would instruct an AI to build an iOS app named TaskFlow using SwiftUI and MVVM. The generated prompt must be complete and copy-pasteable so that running it produces the full app.


Include the following in the generated prompt:

App overview:
Task and habit tracker: create and organize tasks (due dates, priorities, categories, optional reminders, optional tags) and track daily habits with repeat schedules, check-ins, streaks, and optional daily reminders. Single-user, offline-first.


Problem to solve:
Simple, offline-first, single-user workflow; no backend or login; all data on device; clean and easy to use.


Core features
- Tasks: Add/Edit (title, notes, due date, priority, category, reminder at or before due time, multiple tags); filters (All / Today / Overdue / Completed); filter by category; mark complete, delete; local notifications with snooze (15 min, 1 hr, tomorrow).
- Categories: CRUD; color (hex) and SF Symbol icon per category; one category per task.
- Tags: Optional multiple tags per task; many-to-many; create from picker; show as #name on rows.
- Habits: Daily habits with "done today" check-in and streak; repeat on chosen weekdays (or every day); optional daily reminder at set time on those days; add/edit/delete; row shows "Not scheduled today" when not due.
- Insights: Today summary (completed, overdue, due today); week summary; completed-per-day chart (e.g. last 14 days); productivity by category.
- Settings: Appearance (System / Light / Dark, persisted app-wide), About (name, version, description).

Technical requirements:
SwiftUI, MVVM, SwiftData (Task, Category, Priority, Tag, Habit, HabitCheckIn), iOS 17+, dark mode (system default + Settings override), shared theme (e.g. TaskFlowTheme), Combine or Swift Concurrency. Require one type per file (filename = type name) and MARK comments for navigation. Note: avoid naming clash between SwiftData model Task and Swift's Task (e.g. use _Concurrency.Task when spawning async). Notification permission requested before scheduling; show notifications in foreground via delegate.

UI structure:
Tab bar: Tasks | Habits | Insights | Categories | Settings. Specify: Home (task list with filter chips and category filter), Add/Edit Task, Habits list and add/edit, Insights (today, week, chart, by category), Categories (with color/icon), Settings. Mention card-style list, FAB for new task, filter UI.

Output Expectations - Provide Data Models, ViewModels, SwiftUI Views, unit tests, with comments. Structured for MVVM but expandable for future features.
```

**Outcome:** The AI returned a full Cursor build prompt, which we copied into Cursor (and optionally refined once).

---

### Step 3: Build TaskFlow in Cursor (full prompt)

**What we did:** Pasted the generated prompt into Cursor to create the app with all agreed features in one go. All code in this repo was produced by Cursor from this (or follow-up) prompts.

**Prompt used (representative):**

```
You are a senior iOS engineer specializing in SwiftUI and MVVM architecture.

Build an iOS mobile app named TaskFlow using SwiftUI.
TaskFlow is a task and habit tracker that allows users to create, organize, and complete tasks with due dates, categories, and priorities. Optionally, users can track simple daily habits (e.g., "Exercise", "Read") with check-ins and streaks.

Problem to Solve:

Users need a simple way to capture tasks, set due dates, and organize them by category and priority. Existing apps are either too heavy or lack a focused flow for quick personal use. TaskFlow is single-user, offline-first—no backend or login; all data stays on the device.

Core Features:

1. Add Task
   - Title (required), optional notes
   - Due date and time
   - Priority (Low, Medium, High)
   - One category per task
   - Optional reminder: at due time or 5 / 15 / 30 min or 1 hour before (local notifications)
   - Optional multiple tags per task (e.g. #urgent); many-to-many; create tags from picker

2. Task Organization
   - Clean list with filter chips: All / Today / Overdue / Completed
   - Filter by category (toolbar menu: "All categories" or specific category)
   - Group by Today / Overdue / Upcoming / Completed
   - Card-style rows with checkbox, category badge, tags, due/overdue

3. Categories
   - Create, edit, delete (e.g. Work, Personal, Health)
   - Each category can have a color (hex) and SF Symbol icon
   - One category per task

4. Task Actions
   - Tap task to view or edit (title, notes, due date, priority, category)
   - Mark task complete (checkbox or toggle)
   - Delete task
   - Save changes

5. Reminders and notifications
   - Optional reminder per task (at due time or N minutes before)
   - Local notifications; request permission when user sets a reminder or taps Save with reminder on
   - Snooze actions: 15 min, 1 hour, tomorrow 9:00 AM
  - Request permission before scheduling; show notifications in foreground/background via delegate

6. Habits
   - Daily habits (e.g. Exercise, Read) with "done today" check-in and streak display
  - Repeat schedule: which weekdays (e.g. Mon, Wed, Fri) or every day; empty = every day
  - Daily reminder at a set time on scheduled days only (repeating calendar triggers per weekday)
  - Add/Edit: name, repeat weekdays (picker), optional reminder time; context menu: Edit, Delete
  - Rows show "Every day" or "Mon, Wed, Fri"; toggle disabled with "Not scheduled today" when not due today

7. Tags
   - Tag model (id, name); tasks have optional multiple tags; one-way relationship from Task to Tag if needed to avoid SwiftData macro issues
  - Tag multi-select + "Add tag" in task edit; display as #name on task rows

8. Insights
  - Today: completed today, overdue, due today
  - This week: completed count
  - Completed per day: chart for last 14 days (Swift Charts)
  - By category: completion/productivity per category

9. Empty States
    - Each major screen must include thoughtful empty states. EX -"No tasks yet. Tap + to create your first task."

10. Settings
- Appearance: System / Light / Dark (persisted, applied app-wide)
- About: app name, version/build, short description

Technical Requirements:
- SwiftUI, MVVM, SwiftData for persistence
- iOS 17+
- Dark mode: respect system by default; Settings allows System / Light / Dark
- Clean, modern UI; follow Apple HIG
- Use Combine or Swift Concurrency where appropriate
- Production-ready, modular, easy to extend
- One type per file: one primary type per Swift file; filename = type name
- MARK comments: use // MARK: - for Properties, Body, Subviews, Public Methods, etc.

UI Screens:
- Tab bar: Tasks | Habits | Insights | Categories | Settings
- Tasks: list (filter chips + category filter in toolbar), FAB to add task, card rows
- Task add/edit: form with all fields; Save / Delete
- Habits: list with weekday repeat and reminder in add/edit; streak and "done today" on row
- Insights: today, week, chart, by category
- Categories: CRUD with color and icon picker
- Settings: Appearance, About

Extra Notes:
- Use Combine or Swift Concurrency for data binding
- Follow Apple Swift, SwiftUI and Interface Guidelines docs
- Ensure the codebase is production-ready and easy to extend

Build and run:
- Prefer XcodeBuild MCP: set session defaults (project path, scheme, simulator name), then build and run on simulator (e.g. iPhone 15, iPhone 16 Pro, iPhone 17 Pro). If a requested simulator (e.g. "iPhone 17 Pro Max") is not installed, use the closest available (e.g. iPhone 17 Pro or iPhone 16 Pro Max).

Output Expectations:
Data models (Task, Category, Priority, etc.), ViewModels, SwiftUI views
Unit tests for ViewModels and models where practical
Comments for non-obvious decisions
Optional: HTML UI visualization file for stakeholders
```

**Outcome:** Cursor generated the full app: project structure, models, list/detail/add screens, persistence, notifications, habits, insights, and the features above. Follow-up prompts were used only for polish or small fixes—no manual coding.
